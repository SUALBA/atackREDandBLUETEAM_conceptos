🛡️ Reglas Locales de Snort: Defensa con Estilo 🚀

Guía visual e intuitiva para detectar ataques comunes

Regla 1: Detectar escaneo Nmap 🕵️ 

alert tcp any any -> $HOME_NET any 
(msg: "NMAP Scan Detected"; sid:1000001; flags:S; 
threshold:type threshold, track by_dst, count 5, seconds 60)

📖 Explicación: Detecta múltiples paquetes SYN → típico de Nmap.

⚡ Consecuencias: Un atacante mapea tus servicios, primer paso de intrusión.

🎯 Conclusión: Verifica servicios expuestos y aplica firewall + IDS.

Regla 2: Acceso a .git expuesto 🔑 

alert tcp any any -> $HOME_NET 80 
(msg:"Access to hidden folder"; sid:1000002; 
content:"/.git/"; http_uri; classtype:web-application-attack;)

📖 Busca accesos a la carpeta .git/ expuesta.

⚡ Riesgo de robo de código fuente, contraseñas y configuraciones.

🎯 Bloquea siempre .git con .htaccess o Nginx.

Regla 3: Login FTP anónimo 🚨 

alert tcp any any -> $HOME_NET 21 
(msg:"Anonymous FTP access"; content:"USER anonymous"; 
classtype:suspicious-login; sid:10000003; rev:1;)

📖 Detecta login FTP con usuario anonymous.

⚡ Riesgo de descarga o subida no autorizada.

🎯 Usa SFTP o usuarios autenticados, nunca FTP anónimo.

Regla 3b: Subida de archivos por FTP 💾 

alert tcp any any -> $HOME_NET 21 
(msg:"Try to upload a file"; content:"STOR"; sid:1000006;)

📖 Detecta comando STOR (subida FTP).

⚡ Riesgo de webshells o malware.

🎯 Desactiva subida si no es necesaria, monitorea actividad.

Regla 4: Fuerza Bruta SSH 🔐 

alert tcp any any -> $HOME_NET 22 
(msg:"Potential SSH Brute Force"; sid:1000004; flow:to_server; flags:S; 
detection_filter:track by_src, count 2, seconds 60; classtype:attempted-dos;)

📖 Detecta múltiples intentos SSH desde una IP.

⚡ Indica fuerza bruta → riesgo de acceso no autorizado.

🎯 Usa fail2ban o bloqueos automáticos.

Regla 5: Inyección de comandos en puerto 5011 ⚔️ 

alert tcp any any -> any 5011 
(msg:"Posible inyección de comandos por el puerto 5011"; 
content:"|3B|"; sid:1000008;)

📖 Busca el carácter ; usado en inyecciones.

⚡ Permite ejecutar múltiples comandos.

🎯 Escapa entradas y audita servicios en el puerto 5011.

🌈 Reflexiones Finales

Estas reglas convierten tu IDS en un perro guardián digital 🐕‍🦺.

Detectan escaneos, accesos indebidos y ataques comunes, dándote tiempo de reaccionar antes del daño real.
