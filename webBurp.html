1. Protocolo HTTP

HTTP (HyperText Transfer Protocol) es el lenguaje que utilizan los navegadores y los servidores para intercambiar recursos mediante un modelo «petición → respuesta». Opera típicamente sobre el puerto 80 (HTTP) o 443 (HTTPS).

2. Métodos HTTP

Los 4 de uso diario

  • GET → Solicita un recurso (idempotente).
  • POST → Envía datos para crear/procesar.
  • PUT → Reemplaza un recurso existente.
  • DELETE → Elimina un recurso.

Verbos complementarios

HEAD, OPTIONS, PATCH y TRACE se emplean para meta‑información, introspección o actualizaciones parciales.

3. Partes de una petición HTTP

GET /login HTTP/1.1
Host: example.com
User-Agent: Firefox/126.0
Cookie: session=abc123
  • Línea de inicio: método, ruta y versión.
  • Cabeceras: pares clave-valor (Host, Cookie, Accept…).
  • Cuerpo: solo para métodos que envían datos (POST, PUT…).

4. Ataques a aplicaciones web

Las vulnerabilidades se explotan enviando payloads que fuerzan comportamientos imprevistos.

Payload Box (diccionario de payloads)

Colecciones como PayloadAllTheThings y el plugin «Burp Payload Box» reúnen cadenas para XSS, SQLi, SSTI, etc., listas para pegar en pruebas.

5. ¿Qué es una aplicación web?

Estáticas vs Dinámicas

  • Estáticas: HTML/CSS pre‑generado, cambia solo al desplegar.
  • Dinámicas: HTML generado al vuelo (PHP, Node.js, Django…).

Pilas full‑stack populares

  • LAMP: Linux · Apache · MySQL · PHP.
  • MEAN: MongoDB · Express · Angular · Node.
  • MERN: MongoDB · Express · React · Node.

6. Herramientas esenciales

Navegadores de desarrollo
  • Chrome DevTools: inspección, Network, Lighthouse.
  • Firefox Developer Edition: pestaña Storage, herramientas CSS.
Enumeración de rutas
  • Dirbuster / Dirb → fuerza bruta de directorios.
Escáneres de vulnerabilidades
  • Nikto: tests HTTP peligrosos.
  • OWASP ZAP: proxy + escáner activo.
Burp Suite

Proxy, Repeater, Intruder, Decoder y extensiones para investigación avanzada.

7. Instalar certificado de Burp en el navegador

  1. Con Burp en marcha, navega a http://burpsuite.
  2. Descarga el certificado cacert.der.
  3. Firefox: Settings → Privacy & Security → View Certificates → Import…
  4. Marca “Confiar en esta CA para identificar sitios web”.
  5. Reinicia el navegador y prueba interceptar un sitio HTTPS.
