rev:1 → cuando creas una regla nueva

rev:2, rev:3, etc. → cada vez que la editas y mejoras

No cambies el SID (es el identificador único), pero sí incrementa rev para llevar control de cambios.
📌 Buenas prácticas con rev
🔹 Empieza siempre en rev:1
(como hice en tu set inicial).

🔹 Incrementa el número cada vez que modifiques la regla:

cambias la condición (ej. de any any a un puerto específico)

añades opciones como content, flags, dsize, etc.

mejoras el mensaje o haces la detección más fina

🔹 Lleva un registro
Puedes añadir un comentario en el local.rules:


# Revisión 2: mejorada para solo ICMP Echo Request
alert icmp any any -> any any (msg:"ICMP Echo Request detectado"; icmp_type:8; sid:2000101; rev:2;)
📌 Resumen
Todas tus reglas están en rev:1 porque son las primeras versiones.

Cuando las modifiques (más específicas, con nuevos patrones, etc.),
aumenta el número de rev.

Esto te permite mantener historial de cambios sin perder el SID original.

# ========================
# Ejemplo SQL Injection
# ========================

# rev:1 → Detección básica de SELECT
alert tcp any any -> any 80 (msg:"SQL Injection detectada (SELECT)"; content:"SELECT"; nocase; sid:2000501; rev:1;)

# rev:2 → Añadimos UNION para ampliar cobertura
alert tcp any any -> any 80 (msg:"SQL Injection detectada (SELECT o UNION)"; content:"UNION"; nocase; sid:2000501; rev:2;)

# rev:3 → Mejoramos con condición múltiple (OR)
alert tcp any any -> any 80 (msg:"SQL Injection avanzada detectada"; content:"SELECT"; nocase; sid:2000501; rev:3;)
alert tcp any any -> any 80 (msg:"SQL Injection avanzada detectada"; content:"UNION"; nocase; sid:2000501; rev:3;)

# ========================
# Ejemplo ICMP Ping
# ========================

# rev:1 → Básico
alert icmp any any -> any any (msg:"ICMP Ping detectado"; sid:2000101; rev:1;)

# rev:2 → Más específico: solo Echo Request
alert icmp any any -> any any (msg:"ICMP Echo Request detectado"; icmp_type:8; sid:2000101; rev:2;)

# rev:3 → Incluimos tamaño mínimo de paquete
alert icmp any any -> any any (msg:"ICMP Echo Request sospechoso (payload grande)"; icmp_type:8; dsize:>100; sid:2000101; rev:3;)
